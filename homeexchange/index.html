HomeExchange
Customer Behaviour & Retention Analysis

HomeExchange is a home-swap travel platform where members pay an annual fee of €149 to access peer-to-peer house exchanges. The goal of this project is to understand why members do not renew their subscription and identify actionable levers to reduce churn.

Business context

How HomeExchange works

Registration is free. The €149 annual membership only applies once a member secures their first exchange. This creates a distinct behavioural split: some users subscribe and never exchange (particularly American users), while European users tend to validate the system before committing.

This cultural difference is a key variable in the churn analysis — churn drivers are not uniform across geographies.

Potential churn drivers
User tenure · guest & host activity · house location · sponsorship · promotional discounts

Data model

Two source tables

The subscriptions table contains one row per subscription event per user. Users who renew appear multiple times. The exchanges table contains conversation and exchange records — including reciprocal exchanges, which require careful counting to avoid double-counting.

Key data quality issues found

  • 276 duplicate rows in the subscriptions table (same user, same subscription year)
  • Inconsistent home_type casing — "Home" vs "apartment"
  • Null values in country, region, city across both tables
  • Reciprocal exchanges create two exchange rows per conversation — requires deduplication logic
  • guest_count = 0 on several exchange rows despite completed exchanges
Data cleaning — subscriptions

Deduplicated customer view

The core challenge was normalising multiple subscription rows per user into a single customer record, preserving all relevant history. The query below produces one row per customer with engineered tenure, churn, and segment features.

BigQuery SQL · customer_base.sql ▾ show query 
WITH sub AS (
  SELECT
    user_id,
    first_subscription_date        AS start_subscription_date,
    MIN(subscription_date)          AS first_filed_subscription_date,
    MAX(subscription_date)          AS last_subscription_date,
    MAX(referral)                   AS total_referrals,
    SUM(promotion)                  AS total_promotions,
    MAX(payment3x)                  AS ever_used_payment3x,
    COUNT(DISTINCT EXTRACT(
      YEAR FROM subscription_date))  AS total_renewals_raw,
    country, region, department, city
  FROM home_exchange.subscriptions
  GROUP BY
    user_id, first_subscription_date,
    country, region, department, city
)

SELECT
  user_id,
  start_subscription_date,
  first_filed_subscription_date,
  last_subscription_date,
  total_referrals,
  total_promotions,
  ever_used_payment3x,

  -- Tenure: years as a member
  DATE_DIFF(
    DATE(last_subscription_date),
    DATE(start_subscription_date), YEAR
  ) AS tenure_years,

  -- Renewals: years active minus starting year
  DATE_DIFF(
    DATE(last_subscription_date),
    DATE(first_filed_subscription_date), YEAR
  ) AS total_renewals,

  -- Churn flag: no renewal in past 13 months
  CASE
    WHEN DATE_DIFF(
      CURRENT_DATE(),
      DATE(last_subscription_date), MONTH
    ) > 13 THEN 1
    ELSE 0
  END AS is_churned,

  -- Tenure bucket for segmentation
  CASE
    WHEN DATE_DIFF(..., YEAR) = 0
      THEN '0 – New'
    WHEN DATE_DIFF(..., YEAR) BETWEEN 1 AND 2
      THEN '1–2 years'
    WHEN DATE_DIFF(..., YEAR) BETWEEN 3 AND 5
      THEN '3–5 years'
    ELSE '6+ years'
  END AS tenure_bucket,

  country, region, department, city
FROM sub

Why this approach

  • MAX(referral) — if a user was ever referred, flag it regardless of which year
  • SUM(promotion) — captures total promotions received across all subscriptions
  • DATE_DIFF(..., YEAR) replaces a fragile CASE WHEN EXTRACT(year) = 2021 fix — works correctly for all cohorts
  • 13-month churn threshold accounts for annual billing + grace period

Promotions reduce churn from 46.8% to just 7%. Referral alone cuts it to 34.2%. Combined, they produce 4.9% churn — the lowest in the entire dataset. These are the clearest, most actionable levers in the analysis.
